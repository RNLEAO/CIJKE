# 第一阶段：IMU660RB 与编码器安全诊断

## 本版本用途

本版本用于在电机完全禁止运行的条件下验证：

- IMU660RB 是否初始化成功。
- 平放、文字朝上时，车辆左右旋转是否主要对应 Z 轴。
- 左右编码器原始脉冲、方向电平和有符号计数是否正确。
- 无屏幕时，无线串口是否能完整显示诊断状态。

本版本默认状态：

```text
MOTOR=LOCKED
ELEMENTS=OFF
RUNLOCK=1
```

即使长按实体按键或发送 `RUN`，电机也不会被允许启动。

## 烧录文件

```text
Project\mdk\out_file\CIJIANKE_EM.hex
```

## 无线串口

```text
波特率：115200
数据位：8
停止位：1
校验位：无
流控：无
HEX 发送：关闭
自动发送：关闭
```

## 上电要求

1. 驱动板电源保持关闭，或者断开电机供电。
2. 车辆静止放平，IMU660RB 文字朝上。
3. 上电后的前 3 秒不要碰动车辆，程序会采集陀螺仪零偏。
4. 串口应显示：

```text
BOOT: IMU660RB=OK AXIS=Z MOTOR=LOCKED ELEMENTS=OFF
```

如果显示 `IMU660RB=ERROR`，不要进行后续测试。

## 串口命令

```text
STATUS
STOP
CLEAR
IMU CAL
ELEMENTS OFF
```

`RUN` 在本版本中会返回锁定错误，不会启动电机。`ELEMENTS ON` 同样被拒绝。

## IMU 静态检查

车辆静止时应满足：

- `IMU: STATE=OK`
- `TURN_X10` 接近 0，允许小幅波动。
- `GX_X10/GY_X10/GZ_X10` 不能固定为异常满量程。

如果上电时碰动了车辆，重新放稳后发送：

```text
IMU CAL
```

校准期间不要移动车辆。

## IMU 方向检查

1. 电机供电仍保持关闭。
2. 手动将整车向左旋转约 90 度，记录一帧 IMU 数据。
3. 手动将整车向右旋转约 90 度，记录一帧 IMU 数据。
4. 正常情况下 `GZ_X10` 和 `TURN_X10` 应明显变化，并且左右旋转符号相反。

## 编码器手转检查

1. 分别手动向车辆前进方向转动左轮和右轮。
2. 左轮转动时，`LRAW` 应变化，`RRAW` 应接近 0。
3. 右轮转动时，`RRAW` 应变化，`LRAW` 应接近 0。
4. 同一方向连续转动时，`LSIGNED/RSIGNED` 的符号应稳定，不应反复正负跳变。

## 本阶段通过条件

- IMU660RB 状态为 `OK`。
- 左右旋转主要由 Z 轴响应，方向符号相反。
- 左右编码器通道没有串线。
- 同一手转方向下有符号计数方向稳定。
- `LPWM/RPWM/LAPPLIED/RAPPLIED` 始终为 0。

通过这些条件后，下一阶段才会开放单轮、限时、低 PWM 的开环测试。
